# HW_3 «E-commerce Admin»

---

## 📌 Постановка задачи

Разработать **SPA** административную панель для e-commerce системы, покрывающую полный пользовательский цикл:

1. Регистрация / авторизация пользователя
2. Доступ к защищенным разделам приложения
3. Работа с каталогом продуктов
4. Управление настройками интерфейса
5. Завершение сессии пользователя

Приложение должно быть реализовано с использованием `React` + `TypeScript`, `Redux Toolkit`, `RTK Query`, `React Router` и `i18n`.

В качестве backend API используется [DummyJSON](https://dummyjson.com/docs).

---

## ✨ Функциональные требования

### 1.1 Аутентификация и авторизация

**API:**
- `POST /auth/login`
- `GET /auth/me`

**Функциональность:**
- Страница логина (`/login`)
- Валидация формы авторизации
- Авторизация по логину и паролю
- Получение данных текущего пользователя
- Хранение access token
- Logout (очистка сессии)

**Требования:**
Авторизация реализована с помощью RTK Query
- Данные пользователя хранятся в Redux (auth slice)
- Защищенные маршруты (без авторизации доступ запрещен)
- Автоматическая инициализация пользователя при перезагрузке страницы
- Регистрация может быть реализована как UI-заглушка (DummyJSON не поддерживает полноценную регистрацию).



### 1.2 Маршрутизация и layout
**Публичные маршруты:**
- `/login` — авторизация
- `/register` — регистрация (заглушка)

**Приватные маршруты:**
- `/` — Dashboard
- `/products` — список продуктов
- `/products/:id` — страница продукта
- `/profile` — профиль пользователя
- `/settings` — настройки
- `/logout` — выход из системы
- `*` — 404

**Требования:**
- Разделение публичных и приватных маршрутов
- Protected routes
- Общий layout для приватной части
- Lazy loading страниц

Редиректы:
- неавторизованный → `/login`
- авторизованный → `/`

### 1.3 Работа с продуктами
**API:**
- `GET /products`
- `GET /products/{id}`
- `GET /products/search`

**Функциональность:**
- Список продуктов
- Просмотр детальной страницы продукта
- Поиск по названию
- Пагинация (limit / skip)
- Отображение цены, рейтинга, категории

**Требования:**
- Все запросы через RTK Query
- Использование query-параметров
- Кэширование и повторное использование данных
- Обработка loading / error / empty state

### 1.4 Профиль пользователя
**Использовать:**
- `GET /auth/me`

**Функциональность:**
- Отображение данных пользователя:
  - имя
  - email
- Возможность выхода из системы

### 1.5 Настройки приложения
**Настройки:**
- Язык интерфейса (ru / en)
- Тема (light / dark)
- Размер страницы каталога

**Требования:**
- Хранение настроек в Redux
- Persist (localStorage)
- Мгновенное применение изменений
- Связь с i18n

### 1.6 Интернационализация (i18n)
**Обязательно:**
- Русский и Английский языки
- Перевод:
  - всех страниц
  - форм авторизации
  - ошибок API
  - пустых состояний

**Требования:**
- JSON-файлы переводов
- Переключение без перезагрузки
- Единый слой для работы с переводами

## Архитектура

* TypeScript обязателен
* Feature-based архитектурный подход ([Feature Sliced Design](https://feature-sliced.design/))
* Явное разделение:
  * pages
  * widgets
  * features
  * entities
  * shared
* Отсутствие прямых импортов между слоями
* API-слой изолирован
* Переиспользуемые UI-компоненты
* Бизнес-логика вынесена из UI

```
src/
 ├── app/
 ├── pages/
 ├── widgets/
 ├── features/
 ├── entities/
 └── shared/
```
